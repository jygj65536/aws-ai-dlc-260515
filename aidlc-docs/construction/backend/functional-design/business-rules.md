# Business Rules - Backend

## 1. 인증/인가 규칙

| ID | 규칙 | 조건 | 결과 |
|----|------|------|------|
| AUTH-01 | 관리자 로그인 시도 제한 | 연속 5회 실패 | 15분 계정 잠금 |
| AUTH-02 | 관리자 JWT 만료 | 토큰 발급 후 16시간 경과 | 자동 로그아웃 (401) |
| AUTH-03 | 테이블 JWT 만료 | 토큰 발급 후 16시간 경과 | 재로그인 필요 |
| AUTH-04 | 관리자 전용 API | role != "admin" | 403 Forbidden |
| AUTH-05 | 테이블 전용 API | role != "table" | 403 Forbidden |
| AUTH-06 | 매장 격리 | 요청의 store_id != 토큰의 store_id | 403 Forbidden |

## 2. 주문 상태 전이 규칙

```
         ┌─────────────────────────┐
         │                         │
         v                         │
    [pending] ──→ [preparing] ──→ [completed]
         │                         
         └─────────────────────────→ [completed] (건너뛰기 허용)
```

| ID | 규칙 | 허용 | 비고 |
|----|------|:----:|------|
| ORD-01 | pending → preparing | ✅ | 정상 흐름 |
| ORD-02 | preparing → completed | ✅ | 정상 흐름 |
| ORD-03 | pending → completed | ✅ | 건너뛰기 허용 |
| ORD-04 | completed → * | ❌ | 완료 후 변경 불가 |
| ORD-05 | preparing → pending | ❌ | 역방향 불가 |
| ORD-06 | 삭제 | ✅ | 관리자만, 모든 상태에서 가능 |

## 3. 세션 관리 규칙

| ID | 규칙 | 조건 | 결과 |
|----|------|------|------|
| SES-01 | 세션 자동 생성 | 첫 주문 시 활성 세션 없음 | 새 세션 생성 (4시간 만료) |
| SES-02 | 세션 자동 만료 | expires_at < 현재시각 | status="expired", Lazy 처리 |
| SES-03 | 만료 후 주문 | 세션 만료 상태에서 주문 시도 | 새 세션 자동 생성 |
| SES-04 | 이용 완료 | 관리자가 수동 종료 | 주문→이력 이동, 테이블 리셋 |
| SES-05 | 중복 세션 방지 | 테이블당 활성 세션 | 최대 1개만 허용 |
| SES-06 | 세션 없는 테이블 | current_session_id = null | 주문 내역 비어있음 |

## 4. 데이터 검증 규칙

### 4.1 메뉴 등록/수정

| ID | 필드 | 규칙 | 에러 메시지 |
|----|------|------|-------------|
| VAL-01 | name | 필수, 1~50자 | "메뉴명은 1~50자여야 합니다" |
| VAL-02 | price | 필수, 100 ≤ price ≤ 1,000,000 | "가격은 100원~1,000,000원이어야 합니다" |
| VAL-03 | description | 선택, 최대 200자 | "설명은 200자 이내여야 합니다" |
| VAL-04 | category_id | 필수, 존재하는 카테고리 | "유효하지 않은 카테고리입니다" |
| VAL-05 | sort_order | 선택, 0 이상 정수 | "순서는 0 이상이어야 합니다" |

### 4.2 카테고리 등록/수정

| ID | 필드 | 규칙 | 에러 메시지 |

| VAL-07 | sort_order | 선택, 0 이상 정수 | "순서는 0 이상이어야 합니다" |

### 4.3 주문 생성

| ID | 필드 | 규칙 | 에러 메시지 |

## 5. 비즈니스 정책 규칙

| ID | 규칙 | 설명 |
|----|------|------|
| BIZ-01 | 주문 번호 순번 | 매장별 순차 증가 (1, 2, 3...) |

| BIZ-05 | 메뉴 삭제 시 기존 주문 | 기존 주문의 메뉴 정보는 유지 (스냅샷) |
| BIZ-06 | SSE 2초 이내 전달 | 주문 생성 → SSE 이벤트 발행 즉시 |

## 6. 에러 응답 규칙

| HTTP 코드 | 의미 | 사용 시점 |
|:---------:|------|-----------|
| 400 | Bad Request | 검증 실패, 잘못된 상태 전이 |
| 401 | Unauthorized | 토큰 없음/만료/유효하지 않음 |
| 403 | Forbidden | 권한 없음 (role 불일치, 매장 불일치) |
| 404 | Not Found | 리소스 없음 |
| 423 | Locked | 계정 잠금 |
| 500 | Internal Error | 서버 내부 오류 |

에러 응답 형식:
```json
{
  "detail": "에러 메시지 (한국어)"
}
```
