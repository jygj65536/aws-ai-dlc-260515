# 사용자 스토리 (User Stories)

**분류 방식**: Feature-Based  
**세분화 수준**: Epic  
**AC 형식**: Given/When/Then  
**우선순위**: P1 (필수) / P2 (중요) / P3 (선택)

---

## US-1: 메뉴 조회 및 탐색

**As a** 고객 (김민수)  
**I want to** 매장의 메뉴를 카테고리별로 탐색하고 상세 정보를 확인할 수 있다  
**So that** 원하는 메뉴를 빠르게 찾아 주문할 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-1.1**: 메뉴 목록 표시
- **Given** 고객이 태블릿에서 메뉴 화면에 접근했을 때
- **When** 페이지가 로드되면
- **Then** 카테고리별로 분류된 메뉴 목록이 카드 형태로 표시된다 (메뉴명, 가격, 설명 포함)

**AC-1.2**: 카테고리 이동
- **Given** 메뉴 화면이 표시된 상태에서
- **When** 고객이 다른 카테고리를 선택하면
- **Then** 해당 카테고리의 메뉴만 필터링되어 표시된다

**AC-1.3**: 메뉴 상세 보기
- **Given** 메뉴 목록이 표시된 상태에서
- **When** 고객이 특정 메뉴 카드를 터치하면
- **Then** 메뉴의 상세 정보(메뉴명, 가격, 설명)가 표시된다

**AC-1.4**: 기본 화면
- **Given** 고객이 태블릿에 자동 로그인된 상태에서
- **When** 앱이 시작되면
- **Then** 메뉴 화면이 기본 화면으로 표시된다

## US-2: 장바구니 관리

**As a** 고객 (김민수)  
**I want to** 주문할 메뉴를 장바구니에 담고 수량을 조절할 수 있다  
**So that** 주문 전에 선택한 메뉴를 확인하고 수정할 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-2.1**: 메뉴 추가
- **Given** 메뉴 화면에서 메뉴를 선택한 상태에서
- **When** 고객이 장바구니에 추가 버튼을 터치하면
- **Then** 해당 메뉴가 장바구니에 추가되고 총 금액이 업데이트된다

**AC-2.2**: 수량 조절
- **Given** 장바구니에 메뉴가 담긴 상태에서
- **When** 고객이 수량 증가/감소 버튼을 터치하면
- **Then** 수량이 변경되고 총 금액이 실시간으로 재계산된다

**AC-2.3**: 메뉴 삭제
- **Given** 장바구니에 메뉴가 담긴 상태에서
- **When** 고객이 삭제 버튼을 터치하면
- **Then** 해당 메뉴가 장바구니에서 제거되고 총 금액이 재계산된다

**AC-2.4**: 로컬 저장
- **Given** 장바구니에 메뉴가 담긴 상태에서
- **When** 페이지를 새로고침하면
- **Then** 장바구니 내용이 유지된다

---

## US-3: 주문 생성

**As a** 고객 (김민수)  
**I want to** 장바구니의 메뉴를 주문으로 확정할 수 있다  
**So that** 직원 호출 없이 즉시 주문을 넣을 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-3.1**: 주문 확정
- **Given** 장바구니에 1개 이상의 메뉴가 담긴 상태에서
- **When** 고객이 주문 확정 버튼을 터치하면
- **Then** 주문이 서버로 전송되고 주문 번호가 표시된다

**AC-3.2**: 주문 성공 후 처리
- **Given** 주문이 성공적으로 생성된 후
- **When** 5초가 경과하면
- **Then** 장바구니가 비워지고 메뉴 화면으로 자동 리다이렉트된다

**AC-3.3**: 주문 실패 처리
- **Given** 주문 확정 버튼을 터치했을 때
- **When** 서버 오류로 주문이 실패하면
- **Then** 에러 메시지가 표시되고 장바구니 내용은 유지된다

**AC-3.4**: 주문 정보 포함
- **Given** 주문이 생성될 때
- **When** 서버로 전송되면
- **Then** 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID가 포함된다

---

## US-4: 주문 내역 조회

**As a** 고객 (김민수)  
**I want to** 현재 세션의 주문 내역을 확인할 수 있다  
**So that** 이전에 주문한 내용과 상태를 파악할 수 있다

**Priority**: P2

### Acceptance Criteria

**AC-4.1**: 주문 목록 표시
- **Given** 고객이 주문 내역 화면에 접근했을 때
- **When** 페이지가 로드되면
- **Then** 현재 테이블 세션의 주문만 시간 순으로 표시된다

**AC-4.2**: 주문 상세 정보
- **Given** 주문 목록이 표시된 상태에서
- **When** 각 주문 항목을 확인하면
- **Then** 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다

**AC-4.3**: 이전 세션 제외
- **Given** 테이블에서 이전 고객의 이용 완료 처리가 된 후
- **When** 새 고객이 주문 내역을 조회하면
- **Then** 이전 세션의 주문은 표시되지 않는다

---

## US-5: 테이블 자동 로그인 및 세션 관리

**As a** 고객 (김민수)  
**I want to** 별도 로그인 절차 없이 태블릿에서 바로 주문할 수 있다  
**So that** 번거로운 인증 과정 없이 즉시 서비스를 이용할 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-5.1**: 자동 로그인
- **Given** 관리자가 태블릿 초기 설정을 완료한 후
- **When** 태블릿이 재시작되거나 페이지가 새로고침되면
- **Then** 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다

**AC-5.2**: 세션 유지
- **Given** 태블릿이 자동 로그인된 상태에서
- **When** 16시간이 경과하지 않았으면
- **Then** 세션이 유지되어 정상적으로 서비스를 이용할 수 있다

**AC-5.3**: 테이블 세션 만료
- **Given** 테이블 세션이 시작된 후
- **When** 4시간이 경과하면
- **Then** 테이블 세션이 자동 만료되고 새 세션으로 시작할 수 있다

---

## US-6: 실시간 주문 모니터링

**As a** 관리자 (박서연)  
**I want to** 들어오는 주문을 실시간으로 확인하고 상태를 관리할 수 있다  
**So that** 주문을 놓치지 않고 빠르게 처리할 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-6.1**: 실시간 주문 수신
- **Given** 관리자가 주문 모니터링 대시보드에 접속한 상태에서
- **When** 고객이 새 주문을 생성하면
- **Then** 2초 이내에 대시보드에 새 주문이 표시되고 시각적으로 강조된다

**AC-6.2**: 테이블별 그리드 레이아웃
- **Given** 주문 모니터링 대시보드에서
- **When** 주문이 존재하면
- **Then** 테이블별 카드 형태로 표시되며 각 카드에 테이블 번호, 총 주문액, 최신 주문 미리보기가 포함된다

**AC-6.3**: 주문 상태 변경
- **Given** 주문 카드가 표시된 상태에서
- **When** 관리자가 상태 변경 버튼을 클릭하면
- **Then** 주문 상태가 변경된다 (대기중 → 준비중 → 완료)

**AC-6.4**: 주문 상세 보기
- **Given** 테이블 카드가 표시된 상태에서
- **When** 관리자가 카드를 클릭하면
- **Then** 해당 테이블의 전체 주문 메뉴 목록이 상세하게 표시된다

---

## US-7: 테이블 관리

**As a** 관리자 (박서연)  
**I want to** 테이블 설정, 주문 삭제, 이용 완료 처리를 할 수 있다  
**So that** 테이블 운영을 효율적으로 관리할 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-7.1**: 테이블 초기 설정
- **Given** 관리자가 새 테이블을 설정할 때
- **When** 테이블 번호와 비밀번호를 입력하고 저장하면
- **Then** 테이블 세션이 생성되고 해당 태블릿에서 자동 로그인이 가능해진다

**AC-7.2**: 주문 삭제
- **Given** 테이블에 주문이 존재하는 상태에서
- **When** 관리자가 특정 주문의 삭제 버튼을 클릭하고 확인하면
- **Then** 해당 주문이 삭제되고 테이블 총 주문액이 재계산된다

**AC-7.3**: 이용 완료 처리
- **Given** 고객이 식사를 마친 테이블에서
- **When** 관리자가 이용 완료 버튼을 클릭하고 확인하면
- **Then** 해당 세션의 주문이 과거 이력으로 이동하고 테이블이 리셋된다

**AC-7.4**: 과거 내역 조회
- **Given** 관리자가 테이블의 과거 내역을 확인하고 싶을 때
- **When** 과거 내역 버튼을 클릭하면
- **Then** 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다 (날짜 필터링 가능)

---

## US-8: 메뉴 관리

**As a** 관리자 (박서연)  
**I want to** 메뉴를 등록, 수정, 삭제하고 순서를 조정할 수 있다  
**So that** 매장 메뉴를 최신 상태로 유지할 수 있다

**Priority**: P2

### Acceptance Criteria

**AC-8.1**: 메뉴 등록
- **Given** 관리자가 메뉴 관리 화면에서
- **When** 메뉴명, 가격, 설명, 카테고리를 입력하고 저장하면
- **Then** 새 메뉴가 등록되고 고객 화면에 반영된다

**AC-8.2**: 메뉴 수정
- **Given** 기존 메뉴가 등록된 상태에서
- **When** 관리자가 메뉴 정보를 수정하고 저장하면
- **Then** 변경된 정보가 즉시 반영된다

**AC-8.3**: 메뉴 삭제
- **Given** 기존 메뉴가 등록된 상태에서
- **When** 관리자가 삭제 버튼을 클릭하고 확인하면
- **Then** 해당 메뉴가 삭제되고 고객 화면에서 제거된다

**AC-8.4**: 데이터 검증
- **Given** 메뉴를 등록 또는 수정할 때
- **When** 필수 필드가 비어있거나 가격이 유효하지 않으면
- **Then** 검증 에러 메시지가 표시되고 저장이 차단된다

## US-9: 매장 인증

**As a** 관리자 (박서연)  
**I want to** 매장 관리 시스템에 안전하게 로그인할 수 있다  
**So that** 권한 있는 사람만 매장을 관리할 수 있다

**Priority**: P1

### Acceptance Criteria

**AC-9.1**: 로그인
- **Given** 관리자가 로그인 페이지에 접근했을 때
- **When** 매장 ID, 사용자명, 비밀번호를 입력하고 로그인하면
- **Then** 인증이 성공하고 관리자 대시보드로 이동한다

**AC-9.2**: 세션 유지
- **Given** 관리자가 로그인한 상태에서
- **When** 16시간이 경과하지 않았으면
- **Then** 브라우저 새로고침 시에도 세션이 유지된다

**AC-9.3**: 자동 로그아웃
- **Given** 관리자가 로그인한 상태에서
- **When** 16시간이 경과하면
- **Then** 자동으로 로그아웃되고 로그인 페이지로 이동한다

**AC-9.4**: 로그인 시도 제한
- **Given** 로그인 페이지에서
- **When** 연속으로 잘못된 비밀번호를 입력하면
- **Then** 일정 횟수 초과 시 로그인 시도가 제한된다
