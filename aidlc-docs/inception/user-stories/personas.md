# 사용자 페르소나 (User Personas)

---

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 고객) |
| **역할** | 매장 방문 고객 |
| **연령대** | 20~50대 |
| **기술 수준** | 초급~중급 (스마트폰/태블릿 기본 사용 가능) |

### 목표
- 대기 시간 없이 빠르게 주문하고 싶다
- 메뉴를 쉽게 탐색하고 원하는 것을 찾고 싶다
- 주문 상태를 확인하고 싶다

### 특성
- 직원 호출 없이 자율적으로 주문하는 것을 선호
- 복잡한 절차 없이 직관적인 인터페이스를 기대
- 터치 기반 조작에 익숙함
- 주문 실수 시 수정할 수 있기를 원함

### 사용 환경
- 매장 내 테이블에 고정된 태블릿
- Wi-Fi 연결 환경
- 식사 시간 중 사용 (평균 30분~1시간)

### Pain Points
- 직원을 기다리는 시간이 길다
- 메뉴판이 복잡하거나 가격이 잘 안 보인다
- 주문이 제대로 들어갔는지 불안하다

---

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (대표 관리자) |
| **역할** | 매장 운영자/사장님 |
| **연령대** | 30~60대 |
| **기술 수준** | 초급~중급 (기본 PC/모바일 사용 가능) |

### 목표
- 들어오는 주문을 실시간으로 확인하고 처리하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 메뉴를 쉽게 등록하고 관리하고 싶다
- 매장 운영 효율을 높이고 싶다

### 특성
- 바쁜 영업 시간 중에도 빠르게 조작해야 함
- 복잡한 설정보다 간단한 조작을 선호
- 주문 누락이나 실수에 민감
- 테이블 회전율에 관심이 높음

### 사용 환경
- 카운터 또는 사무실의 PC/태블릿
- 영업 시간 내내 모니터링 (8~12시간)
- 여러 테이블을 동시에 관리

### Pain Points
- 주문을 놓치거나 지연되는 것이 걱정
- 테이블별 정산이 복잡하다
- 메뉴 변경 시 모든 테이블에 반영이 번거롭다
- 이전 고객 주문과 현재 고객 주문이 섞이는 것이 걱정
