## 기능 요구사항 (Functional Requirements)

### FR-1: 고객용 기능

#### FR-1.1: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 1회 수행 (매장 ID, 테이블 번호, 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 태블릿 인증 토큰 16시간 유지 (테이블 이용 세션과 별도 — 이용 세션은 4시간 자동 만료)

#### FR-1.2: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명 (이미지는 MVP 제외)
- 카테고리 간 빠른 이동
- 터치 친화적 UI (최소 44x44px 버튼)

#### FR-1.3: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-1.4: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID

#### FR-1.5: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 이용 완료 처리된 주문 제외

### FR-2: 관리자용 기능

#### FR-2.1: 매장 인증
- 매장 ID + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증
- 16시간 세션 유지, 이후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-2.2: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트
- 그리드/대시보드 레이아웃 (테이블별 카드)
- 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

#### FR-2.3: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- **이용 완료**: 확인 팝업 → 주문 내역 과거 이력 이동 → 테이블 리셋
- **과거 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-2.4: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL (MVP에서 미사용)
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-1: 성능
- SSE 기반 실시간 주문 알림 (2초 이내)
- 소규모 동시 접속 지원 (테이블 10개 이하)
- 페이지 로딩 3초 이내

### NFR-2: 가용성
- 로컬/온프레미스 환경에서 안정적 운영
- 네트워크 단절 시 장바구니 로컬 유지

### NFR-3: 사용성
- 터치 친화적 UI (태블릿 최적화)
- 직관적 네비게이션
- 명확한 시각적 피드백 (성공/실패)

### NFR-4: 유지보수성
- 다중 매장 확장 가능한 데이터 모델 (store_id 기반)
- 모듈화된 코드 구조
- API 문서화

### NFR-5: 세션 관리
- 테이블 세션: 4시간 자동 만료
- 관리자 세션: 16시간 유지
- 테이블 태블릿: 자동 로그인 (1회 설정 후)

## 제외 범위 (Out of Scope)

요구사항 제약사항 문서(constraints.md) 기반:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 리포트, 재고, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, 소셜, 지도, 번역)

## 데이터 모델 개요

### 주요 엔티티
1. **Store** - 매장 정보
2. **Table** - 테이블 정보 (매장 소속)
3. **TableSession** - 테이블 세션 (시작/종료 시각, 상태)
4. **Category** - 메뉴 카테고리
5. **MenuItem** - 메뉴 항목
6. **Order** - 주문 (세션 소속)
7. **OrderItem** - 주문 항목 (주문 소속)
8. **AdminUser** - 관리자 계정
9. **OrderHistory** - 과거 주문 이력 (이용 완료 후 이동)

## MVP 범위 요약

**고객용**: 자동 로그인 → 메뉴 조회 → 장바구니 → 주문 생성 → 주문 내역 조회
**관리자용**: 매장 인증 → 실시간 주문 모니터링 → 테이블 관리 → 메뉴 관리
